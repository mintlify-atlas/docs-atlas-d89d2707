## Translation System

### Translation Files

Translations are stored as JSON files in the `i18n/` directory:

```javascript
// From i18n.js:6-7
const FOLDER = 'i18n';
const API_BASE = 'https://gitlab.wikimedia.org/api/v4/projects/kevinpayravi%2Fcite-unseen/repository';
```

Each language has its own JSON file:
- `en.json` - English
- `ja.json` - Japanese  
- `zh-hans.json` - Simplified Chinese
- `zh-hant.json` - Traditional Chinese
- `ar.json` - Arabic
- `ko.json` - Korean
- And 10 more...

### Translation Structure

Translation files use a flat structure with dot-notation keys:

```json
{
  "settingsButton": "Cite Unseen Settings",
  "cancel": "Cancel",
  "save": "Save",
  "categoryHints.news": "This source is from an outlet whose primary purpose is journalistic reporting",
  "categoryLabels.news": "news",
  "citationTooltipPrefix": "From",
  "citationTooltipSuffix": ":"
}
```

### How Translations Are Loaded

The translation system loads and caches all language files:

```javascript
// From i18n.js:92-107
async function buildI18nObject() {
  const files = await listI18nFiles();
  const result = {};
  
  await Promise.all(files.map(async f => {
    const langCode = f.name.replace(/\.json$/i, '');
    try {
      const flat = await fetchJsonFile(`${FOLDER}/${f.name}`);
      mergeLanguageData(result, langCode, flat);
    } catch (e) {
      console.error('[Cite Unseen][i18n] Error fetching language file', f.name, e);
    }
  }));
  
  return result;
}
```

### Translation Caching

Translations are cached in localStorage for 72 hours:

```javascript
// From i18n.js:8-9
const CACHE_KEY = 'CiteUnseenI18nCache-v2.1.6';
const CACHE_TTL_MS = 72 * 60 * 60 * 1000; // 72 hours
```

This reduces network requests and improves performance.

## Language Detection

Cite Unseen automatically detects your Wikipedia language using MediaWiki's configuration:

```javascript
// From main.js:1409-1420
const lang = mw.config.get('wgContentLanguage');
CiteUnseen.convByVar = function (i18nDict) {
  const locale = new Intl.Locale(lang);
  if (locale.language === 'zh') {
    if (locale.script === 'Hans') {
      return i18nDict['hans'] || i18nDict['hant'] || i18nDict['en'] || 'Language undefined!';
    } else {
      return i18nDict['hant'] || i18nDict['hans'] || i18nDict['en'] || 'Language undefined!';
    }
  }
  return i18nDict[lang] || i18nDict['en'] || 'Language undefined!';
};
```

### Fallback Chain

If a translation is missing, Cite Unseen falls back in this order:

1. **Current language** - e.g., `ja` for Japanese Wikipedia
2. **English** (`en`) - Default fallback for all languages
3. **"Language undefined!"** - Error message if no translation exists

### Special Case: Chinese Wikipedia

Chinese Wikipedia uses a special integration with HanAssist for variant conversion:

```javascript
// From main.js:1402-1407
if (mw.config.get('wgServer') === "//zh.wikipedia.org") {
  // On Chinese Wikipedia, prioritize using the ext.gadget.HanAssist module
  await mw.loader.using('ext.gadget.HanAssist', function (require) {
    const { convByVar } = require('ext.gadget.HanAssist');
    CiteUnseen.convByVar = convByVar;
  });
}
```

This provides seamless conversion between Simplified (`hans`) and Traditional (`hant`) Chinese.

## Using Translations

Translations are accessed through the `convByVar` function:

```javascript
// From main.js:689

```

The `CiteUnseenI18n` object contains all loaded translations:

```javascript

```

## Plural Handling

English translations include a special plural marker `(s)` that is dynamically processed:

```javascript
// From main.js:755-757
function parseI18nPlural(string, value) {
  return string.replace(/\(s\)/g, value === 1 ? '' : 's');
}
```

Example usage:

```javascript
// From en.json
{
  "categoryLabels.blogs": "blog post(s)",
  "categoryLabels.press": "press release(s)"
}

// Usage in code
const label = parseI18nPlural(convByVar(CiteUnseenI18n.categoryLabels.blogs), 1);
// Result: "blog post" (singular)

const label = parseI18nPlural(convByVar(CiteUnseenI18n.categoryLabels.blogs), 5);
// Result: "blog posts" (plural)
```

<Note>
The plural handling is currently only implemented for English. Other languages should provide separate singular and plural forms or handle pluralization in their own way.
</Note>

## Contributing Translations

To add or improve translations:

1. **Find the language file** in the `i18n/` directory on GitLab
2. **Copy the structure** from `en.json` as a reference
3. **Translate all keys** while preserving the JSON structure
4. **Test locally** by updating your browser's language settings
5. **Submit via GitLab** or propose changes through Meta-Wiki

### Translation Guidelines

- Keep technical terms consistent (e.g., "dashboard", "filter")
- Match the tone: concise and informative
- Test with actual citations to ensure clarity
- Consider character length for UI elements
- Use gender-neutral language where applicable
